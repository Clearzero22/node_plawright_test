## 编写自动化脚本

创建一个 `.ts` 文件，核心流程分四步：

```typescript
import { chromium } from 'playwright';

(async () => {
  // 1. 启动浏览器（headless: false = 显示浏览器窗口）
  const browser = await chromium.launch({ headless: false });
  const page = await browser.newPage();

  // 2. 导航到目标网页
  await page.goto('https://www.example.com/');

  // 3. 执行操作：点击、输入、截图等
  // await page.click('selector');
  // await page.fill('selector', 'text');
  // await page.screenshot({ path: 'output.png' });

  // 4. 关闭浏览器
  await browser.close();
})();
```

## 运行方式

使用 `tsx` 执行 TypeScript 脚本：

```bash
npx tsx automation_demo.ts
```

> `npx tsx` 会临时安装并运行 tsx，无需手动安装。

## 本次 Demo 执行过程

1. **启动浏览器** — `chromium.launch({ headless: false })` 打开可见窗口
2. **打开 bilibili** — `page.goto('https://www.bilibili.com/')`，获取页面标题

## 常用 API 速查

| API | 用途 |
|-----|------|
| `chromium.launch()` / `.launch({ headless: true })` | 启动浏览器（有窗口/无窗口） |
| `page.goto(url)` | 导航到网页 |
| `page.locator(selector)` | 定位页面元素 |
| `page.click(selector)` | 点击元素 |
| `page.fill(selector, text)` | 在输入框填入文字 |
| `page.press(selector, key)` | 按下键盘按键（如 `Enter`） |
| `page.textContent(selector)` | 获取元素文本 |
| `page.screenshot({ path })` | 截图保存 |
| `page.waitForTimeout(ms)` | 等待指定毫秒 |
| `browser.close()` | 关闭浏览器 |

## 与测试模式的区别

| | 测试模式 | 自动化模式（本文档） |
|---|---|---|
| 导入 | `@playwright/test` | `playwright` |
| 运行 | `npx playwright test` | `npx tsx script.ts` |
| 浏览器管理 | 框架自动管理 | 手动 launch/close |
| 断言 | 内置 `expect` + 自动重试 | 需要自行判断 |
| 适用场景 | 自动化测试 | 爬虫、数据采集、批量操作 |
