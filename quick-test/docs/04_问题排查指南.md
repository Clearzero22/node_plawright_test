# 问题排查指南

本文档提供了常见问题的诊断和解决方案。

## 🔧 常见问题

### 1. Chrome进程冲突

**症状：**
```
Error: browserType.launchPersistentContext: Target page, context or browser has been closed
```

**原因：**
多个测试脚本同时运行，或Chrome进程未完全关闭

**解决方案：**

```bash
# 方案1：手动关闭Chrome进程
taskkill /F /IM chrome.exe

# 方案2：使用串行测试
npm run test:all  # 使用串行测试而非并行

# 方案3：等待进程完全关闭
# 在测试脚本中添加
await sleep(3000);  // 等待3秒
```

**预防措施：**
- 使用串行测试而非并行测试
- 每次测试后确保浏览器完全关闭
- 避免同时运行多个测试实例

---

### 2. 用户数据目录锁定

**症状：**
```
Error: User data directory is already in use
```

**原因：**
另一个Chrome实例正在使用相同的用户数据目录

**解决方案：**

```bash
# 方案1：完全关闭Chrome
taskkill /F /IM chrome.exe
taskkill /F /IM chrome.exe /T

# 方案2：使用不同的用户数据目录
# 修改 utils.ts 中的 getUserDataDir()
export function getUserDataDir(profileName: string = 'chromium-profile'): string {
  const userDataDir = path.join(os.homedir(), 'AppData', 'Local', 'Temp', `chrome-${profileName}-${Date.now()}`);
  return userDataDir;
}

# 方案3：删除锁定文件
# 删除用户数据目录下的 lockfile
rm "C:\Users\admin\AppData\Roaming\node_plawright_test\chromium-profile\lockfile"
```

### 3. 登录状态检测失败

**症状：**
- 总是显示"未登录"
- 明明已登录但检测为未登录

**原因：**
- 检测元素选择器错误
- 页面加载时间不足
- 元素在iframe中
- 动态加载的元素

**解决方案：**

```typescript
// 方案1：增加等待时间
await sleep(5000);  // 等待5秒

// 方案2：等待特定元素
await page.waitForSelector('.user-info', { timeout: 10000 });

// 方案3：使用多个选择器
const userMenu = await page.$('.user-info, .user-dropdown, [class*="user"]');

// 方案4：检查iframe
const frame = page.frame('iframe-name');
const element = await frame.$('.user-info');

// 方案5：使用XPath
const element = await page.$('xpath=//div[contains(@class, "user")]');

// 方案6：检查页面文本
const content = await page.content();
const isLoggedIn = content.includes('用户名');
```

**调试技巧：**

```typescript
// 1. 截图查看页面状态
await page.screenshot({ path: 'debug.png' });

// 2. 查看页面HTML
const html = await page.content();
console.log(html);

// 3. 查找所有可能的元素
const elements = await page.$$('div[class*="user"]');
console.log('Found', elements.length, 'user elements');

// 4. 尝试不同的选择器
const selectors = [
  '.user-info',
  '.user-dropdown',
  '[data-testid="user-menu"]',
  'xpath=//div[contains(@class, "user")]'
];

for (const selector of selectors) {
  const element = await page.$(selector);
  console.log(selector, element ? 'Found' : 'Not found');
}
```

### 4. 页面加载超时

**症状：**
```
Error: Timeout 30000ms exceeded
```

**原因：**
- 网络速度慢
- 页面资源过大
- 服务器响应慢

**解决方案：**

```typescript
// 方案1：增加超时时间
await page.goto(url, {
  timeout: 60000,  // 60秒
  waitUntil: 'domcontentloaded'  // 不等待所有资源
});

// 方案2：使用更宽松的等待策略
await page.goto(url, {
  waitUntil: 'commit'  // 只等待HTML开始加载
});

// 方案3：等待特定元素
await page.goto(url, { waitUntil: 'domcontentloaded' });
await page.waitForSelector('.main-content', { timeout: 30000 });

// 方案4：忽略HTTPS错误
const context = await chromium.launchPersistentContext(userDataDir, {
  ignoreHTTPSErrors: true,
  // ... 其他配置
});
```

### 5. 元素找不到

**症状：**
```
Error: No element found for selector
```

**原因：**
- 选择器错误
- 元素未加载
- 页面结构变化
- 动态内容

**解决方案：**

```typescript
// 方案1：等待元素出现
await page.waitForSelector(selector, { timeout: 10000 });

// 方案2：使用更宽松的选择器
// 原始：'.user-info .avatar img'
// 改进：'.avatar img' 或 'img.avatar'

// 方案3：使用文本内容
const element = await page.$('text=用户名');

// 方案4：使用属性选择器
const element = await page.$('[data-testid="user-avatar"]');

// 方案5：组合选择器
const element = await page.$('.user-info, .user-menu, [class*="user"]');
```

### 6. Cookie/数据未保存

**症状：**
- 每次运行都需要重新登录
- 登录状态不持久

**原因：**
- 用户数据目录配置错误
- Cookie被阻止
- 隐私模式
- 数据目录权限问题

**解决方案：**

```typescript
// 方案1：确认用户数据目录
const userDataDir = path.join(os.homedir(), 'AppData', 'Roaming', 'node_plawright_test', 'chromium-profile');
console.log('User data dir:', userDataDir);

// 方案2：检查目录权限
const fs = require('fs');
if (!fs.existsSync(userDataDir)) {
  fs.mkdirSync(userDataDir, { recursive: true });
}

// 方案3：验证Cookie保存
const context = await launchPersistent();
const cookies = await context.cookies();
console.log('Cookies:', cookies.length);

// 方案4：手动保存状态
await context.storageState({ path: 'auth.json' });

// 方案5：检查网站Cookie策略
// 某些网站可能禁止在自动化浏览器中保存Cookie
```

### 7. 截图失败

**症状：**
```
Error: Screenshot failed
```

**原因：**
- 页面未完全加载
- 元素被遮挡
- 文件权限问题

**解决方案：**

```typescript
// 方案1：等待页面稳定
await page.waitForLoadState('networkidle');
await page.screenshot({ path: 'screenshot.png' });

// 方案2：使用完整页面截图
await page.screenshot({
  path: 'screenshot.png',
  fullPage: true
});

// 方案3：截图特定元素
const element = await page.$('.content');
await element.screenshot({ path: 'screenshot.png' });

// 方案4：确保输出目录存在
const fs = require('fs');
const outputDir = 'output';
if (!fs.existsSync(outputDir)) {
  fs.mkdirSync(outputDir, { recursive: true });
}
```

### 8. TypeScript编译错误

**症状：**
```
Error: Cannot find module 'playwright'
```

**原因：**
- 依赖未安装
- TypeScript配置错误
- 路径配置错误

**解决方案：**

```bash
# 方案1：安装依赖
npm install playwright

# 方案2：安装TypeScript
npm install -D typescript tsx

# 方案3：重新编译
npm run build

# 方案4：清理缓存后重新安装
rm -rf node_modules package-lock.json
npm install
```

### 9. CDP连接失败

**症状：**
```
Error: connect ECONNREFUSED ::1:9222
```

**原因：**
- Chrome未启动CDP模式
- 端口被占用
- 防火墙阻止

**解决方案：**

```bash
# 方案1：启动Chrome CDP模式
chrome.exe --remote-debugging-port=9222

# 方案2：检查端口是否被占用
netstat -ano | findstr :9222

# 方案3：使用其他端口
chrome.exe --remote-debugging-port=9223

# 修改连接代码
await chromium.connectOverCDP('http://localhost:9223');

# 方案4：检查防火墙
# 允许Chrome通过防火墙
```

### 10. 内存泄漏

**症状：**
- 多次运行后变慢
- 内存占用持续增长
- Chrome进程未关闭

**解决方案：**

```typescript
// 方案1：确保正确关闭
const context = await launchPersistent();
const page = await getPageFromContext(context);

try {
  // 测试代码
} finally {
  await page.close();
  await context.close();
}

// 方案2：使用单例模式
let browserContext: BrowserContext | null = null;

async function getContext() {
  if (!browserContext) {
    browserContext = await launchPersistent();
  }
  return browserContext;
}

// 方案3：定期重启
let testCount = 0;
const MAX_TESTS = 10;

if (testCount >= MAX_TESTS) {
  await context.close();
  browserContext = null;
  testCount = 0;
}
```

## 🔍 调试技巧

### 1. 启用详细日志

```typescript
// 在测试脚本开头添加
process.env.DEBUG = 'pw:*';
```

### 2. 使用浏览器开发者工具

```typescript
const context = await launchPersistent();
const page = await getPageFromContext(context);

// 打开开发者工具
// 注意：这会干扰自动化测试
await page.pause();
```

### 3. 慢动作模式

```typescript
// 放慢操作，便于观察
await page.click('button', { delay: 1000 });
```

### 4. 保存页面HTML

```typescript
const html = await page.content();
require('fs').writeFileSync('page.html', html);
```

### 5. 查看网络请求

```typescript
page.on('request', request => {
  console.log('Request:', request.url());
});

page.on('response', response => {
  console.log('Response:', response.url(), response.status());
});
```
