# 开发流程指南

本文档描述了在 `quick-test` 环境中进行功能开发和测试的完整流程。

## 📋 开发流程概述

```
需求分析 → 创建测试脚本 → 快速验证 → 集成到应用 → 完整测试
```

## 🔧 详细开发步骤

### 1. 需求分析

明确要开发的功能：
- 目标网站URL
- 需要自动化的操作
- 登录要求
- 数据提取需求

### 2. 创建测试脚本

**方式A：使用生成器（推荐）**
```bash
npm run generate
```

**方式B：手动创建**
```bash
# 复制模板
cp templates/test-template.ts src/test-your-feature.ts

# 或基于现有脚本修改
cp src/test-chatgpt.ts src/test-your-feature.ts
```

### 3. 编写测试代码

编辑 `src/test-your-feature.ts`：

```typescript
import { launchPersistent, getPageFromContext, log, sleep, checkLoginStatus, screenshot } from './utils';

async function test() {
  log('🚀 开始测试...', 'info');

  // 1. 启动浏览器
  const context = await launchPersistent();
  const page = await getPageFromContext(context);

  // 2. 访问网站
  await page.goto('https://example.com', {
    timeout: 30000,
    waitUntil: 'domcontentloaded'
  });

  // 3. 执行操作
  // TODO: 添加你的操作逻辑

  // 4. 验证结果
  const title = await page.title();
  log('页面标题: ' + title, 'info');

  // 5. 截图保存
  await screenshot(page, 'your-feature');

  // 6. 保持浏览器打开（手动操作）
  await new Promise(() => {});
}

test();
```

### 4. 快速验证

```bash
# 运行单个测试
npm run test:your-feature

# 或直接运行
npx tsx src/test-your-feature.ts
```

### 5. 验证检查点

- [ ] 页面能正常加载
- [ ] 登录状态正确检测
- [ ] 操作执行成功
- [ ] 数据正确保存
- [ ] 截图正常生成

### 6. 手动登录（首次）

1. 在打开的浏览器中手动登录
2. 确认登录状态保存
3. 关闭浏览器
4. 重新运行测试验证登录持久化

### 7. 集成到Electron应用

**步骤1：添加任务配置**

编辑 `electron/main/index.ts`：

```typescript
const TASKS: Record<string, {...}> = {
  'your-feature': {
    url: 'https://example.com',
    channel: 'chrome',
    useUserDataDir: true,
  },
  // ... 其他任务
};
```

**步骤2：添加登录检测**

编辑 `checkLoginStatus()` 函数：

```typescript
async function checkLoginStatus(page: any, url: string) {
  if (url.includes('example.com')) {
    const userMenu = await page.$('.user-info');
    const loginButton = await page.$('a[href*="login"]');
    return !!userMenu && !loginButton;
  }
  // ... 其他检测
}
```

**步骤3：添加UI按钮**

编辑 `electron/renderer/index.html` 和 `app.js`

### 8. 完整测试

```bash
# 1. 测试单个功能
npm run test:your-feature

# 2. 测试所有功能
npm run test:all

# 3. 在Electron应用中测试
npm run electron:dev
```

### 9. 功能文档

在 `features/your-feature.md` 中记录：

```markdown
# Your Feature

## 功能描述
简要描述功能

## URL
https://example.com

## 登录检测逻辑
- 检测元素：`.user-info`
- 未登录元素：`a[href*="login"]`

## 开发状态
- [x] 基础访问
- [x] 登录检测
- [x] 数据持久化
- [ ] 高级功能

## 测试结果
- 测试日期：2024-04-26
- 登录状态：✅ 已登录
- 截图：output/your-feature.png

## 已知问题
- 无
```

### 10. 代码提交

```bash
# 添加文件
git add src/test-your-feature.ts
git add features/your-feature.md
git add electron/main/index.ts
git add electron/renderer/index.html
git add electron/renderer/app.js

# 提交
git commit -m "feat: add your feature automation"

# 推送
git push
```

## ✅ 验收标准

功能开发完成后，应满足以下标准：

1. **功能完整性**
   - [ ] 能访问目标网站
   - [ ] 正确检测登录状态
   - [ ] 执行预期操作
   - [ ] 正确处理错误

2. **数据持久化**
   - [ ] 首次手动登录后，下次自动登录
   - [ ] 用户数据正确保存
   - [ ] 与Electron应用数据同步

3. **代码质量**
   - [ ] 代码格式规范
   - [ ] 有适当的错误处理
   - [ ] 有清晰的日志输出
   - [ ] 有功能文档

4. **测试覆盖**
   - [ ] 快速测试通过
   - [ ] Electron应用测试通过
   - [ ] 有测试报告
