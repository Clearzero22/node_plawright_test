# 测试规范

本文档定义了在 `quick-test` 环境中进行自动化测试的标准和规范。

## 📏 编码规范

### 1. 测试脚本命名

- 格式：`test-{feature-name}.ts`
- 示例：
  - `test-chatgpt.ts`
  - `test-seller-sprite-cn.ts`
  - `test-xiyouzhaoci.ts`

### 2. 函数命名规范

```typescript
// ✅ 好的命名
async function testChatGPT() { }
async function checkLoginStatus() { }
async function navigateToPage() { }

// ❌ 不好的命名
async function test1() { }
async function check() { }
async function go() { }
```

### 3. 日志输出规范

使用统一的日志函数：

```typescript
log('🚀 开始测试...', 'info');        // 信息
log('✅ 测试成功', 'success');        // 成功
log('⚠️  警告信息', 'warning');       // 警告
log('❌ 错误信息', 'error');          // 错误
```

### 4. 错误处理规范

```typescript
try {
  // 测试逻辑
} catch (error: any) {
  log('❌ 测试失败: ' + error.message, 'error');
  process.exit(1);
}
```

### 5. 截图命名规范

- 格式：`{feature-name}.png`
- 位置：`output/` 目录
- 示例：
  - `output/chatgpt.png`
  - `output/seller-sprite-cn.png`

## 🔍 测试用例规范

### 1. 基础测试模板

每个测试脚本应包含以下步骤：

```typescript
async function test() {
  // 1. 启动浏览器
  const context = await launchPersistent();
  const page = await getPageFromContext(context);

  // 2. 访问网站
  await page.goto(url, { timeout: 30000, waitUntil: 'domcontentloaded' });

  // 3. 等待页面加载
  await sleep(2000);

  // 4. 验证页面标题
  const title = await page.title();
  log('页面标题: ' + title, 'info');

  // 5. 检查登录状态
  const isLoggedIn = await checkLoginStatus(page, url);

  // 6. 截图保存
  await screenshot(page, featureName);

  // 7. 关闭浏览器
  await page.close();
  await context.close();
}
```

### 2. 登录状态检测规范

在 `utils.ts` 中添加新的登录检测逻辑：

```typescript
export async function checkLoginStatus(page: Page, url: string): Promise<boolean> {
  try {
    if (url.includes('example.com')) {
      // 已登录标志
      const userMenu = await page.$('.user-info, .user-dropdown');
      // 未登录标志
      const loginButton = await page.$('a[href*="login"], .btn-login');

      // 存在用户信息 且 不存在登录按钮 = 已登录
      return !!userMenu && !loginButton;
    }
    return false;
  } catch {
    return false;
  }
}
```

**检测原则：**
- 优先检测明确的"已登录"标识（用户头像、用户菜单）
- 同时检测"未登录"标识（登录按钮）
- 两者结合判断，提高准确性

### 3. 超时设置规范

```typescript
// 页面加载超时
await page.goto(url, {
  timeout: 30000,              // 30秒
  waitUntil: 'domcontentloaded' // DOM加载完成
});

// 元素等待超时
await page.waitForSelector(selector, { timeout: 10000 });

// 固定等待
await sleep(2000);  // 2秒
```

## 🔄 回归测试

当以下情况发生时，需要执行完整回归测试：

1. 修改了 `utils.ts` 核心函数
2. 更新了 Playwright 版本
3. 修改了用户数据目录配置
4. 添加了新的检测逻辑

回归测试命令：

```bash
# 运行所有测试
npm run test:all

# 检查结果
cat output/test-results.json
```
