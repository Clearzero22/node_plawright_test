  完整分析结果

  问题根源

  你的脚本 每次运行都需要重新登录 的原因是：

  1. automation 目录缺少Gemini登录状态

  test-gemini-listing-generator.ts 默认使用 automation
  目录，但该目录中没有Gemini的登录Cookie和Local Storage数据。

  2. simple 目录可能有登录状态

  simple 目录最近有活动（Last Version 显示），但你的脚本没有使用它。

  数据持久化目录总结

  ~/.node-plawright-test/chrome-profile/
  ├── automation/      # ← test-gemini-listing-generator.ts 使用（无Gemini登录）
  ├── file-upload/     # 其他脚本使用
  ├── simple/          # 可能有Gemini登录状态（应使用此目录）
  └── stealth/         # 隐身模式使用

  解决方案

  推荐方案：修改脚本使用 simple 目录

  修改 test-gemini-listing-generator.ts 第178行：

  // 修改前
  const context = await launchPersistent();

  // 修改后（使用 simple 目录，可能有已有登录状态）
  const context = await launchPersistent('simple');

  备选方案：手动登录一次到 automation 目录

  1. 临时修改脚本让它保持运行：
  const context = await launchPersistent();
  const page = await getPageFromContext(context);
  await page.goto('https://gemini.google.com/app');
  await new Promise(() => {}); // 保持浏览器打开

  2. 手动登录一次
  3. 之后运行时就会自动保持登录状态

  最佳实践：统一命名约定

  建议在所有Gemini相关脚本中统一使用：
  const context = await launchPersistent('gemini-shared');

  这样只需要在一个地方登录，所有脚本都能使用。
