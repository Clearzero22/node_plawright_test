## 🔍 技术实现

### 核心代码模式

```typescript
// 创建单Context
const context = await launchPersistent();

// 创建多个Pages
const page1 = await context.newPage();
const page2 = await context.newPage();
const page3 = await context.newPage();

// 并发加载
await Promise.all([
  page1.goto(url1),
  page2.goto(url2),
  page3.goto(url3)
]);

// 并发操作
const results = await Promise.all([
  page1.evaluate(() => extractData1()),
  page2.evaluate(() => extractData2()),
  page3.evaluate(() => extractData3())
]);
```

### 关键技术点

1. **Promise.all** - 实现真正的并行执行
2. **Context共享** - Cookies和LocalStorage自动共享
3. **独立Pages** - 每个页面独立运行，互不干扰

## 📝 使用建议

### 最佳实践

1. **同账号操作：** 确保所有页面使用相同的登录状态
2. **合理并发数：** 建议3-5个页面，避免过多导致资源紧张
3. **错误处理：** 每个页面的操作都应有独立的错误处理
4. **资源清理：** 操作完成后及时关闭页面

### 注意事项

1. **避免跨账号：** 不要用于多账号操作
2. **监控性能：** 注意内存和CPU使用情况
3. **网络限制：** 注意网站的请求频率限制
4. **页面隔离：** 如需隔离数据，使用不同的Context
