     全局有头/无头浏览器设置

     Context

     用户希望在工作流编辑器顶部工具栏增加一个全局开关，控制所有浏览器节点（GigaB
     2B爬虫、Amazon竞品搜索、Amazon商品详情）的有头/无头模式，而不是逐个节点配置
     。

     方案

     在 Header.tsx 的 Run 按钮旁边增加一个「有头模式」toggle 开关，通过 React
     Context 传递给 WorkflowEngine，在执行时覆盖每个节点的 headless 配置。

     文件变更

     ┌───────────────────────────────────────────────┬──────────────────────────
     ┐
     │                     文件                      │           操作
     │
     ├───────────────────────────────────────────────┼──────────────────────────
     ┤
     │ workflow-editor/src/hooks/useWorkflowState.ts │ 新增 globalHeadless
     │
     │                                               │ state，传递给 engine
     │
     ├───────────────────────────────────────────────┼──────────────────────────
     ┤
     │                                               │ execute() 接收
     │
     │ workflow-editor/src/engine/WorkflowEngine.ts  │ globalConfig
     │
     │                                               │ 参数，合并到 ctx.config
     │
     ├───────────────────────────────────────────────┼──────────────────────────
     ┤
     │ workflow-editor/src/components/Header.tsx     │ Run 按钮旁增加 toggle
     │
     │                                               │ 开关
     │
     └───────────────────────────────────────────────┴──────────────────────────
     ┘

     具体实现

     1. WorkflowEngine.ts — execute() 增加 globalConfig 参数

     async execute(
       nodes, callbacks, startIndex?, nodeConfigs?,
       globalConfig?: Record<string, unknown>,  // 新增
     )

     在循环中构建 ctx.config 时：
     config: { ...nodeConfigs[node.id], ...globalConfig }

     globalConfig 中的值覆盖节点级配置，实现全局控制。

     2. useWorkflowState.ts — 新增 state

     const [globalHeadless, setGlobalHeadless] = useState(true);

     将 globalHeadless 传给 engine.execute()：
     engine.execute(nodes, callbacks, 0, nodeConfigsRef.current, { headless:
     globalHeadless });

     通过 startExecution 的参数或直接在 hook 内部使用。

     3. Header.tsx — 在 Run 按钮左侧增加 toggle

     在 Run/Stop 按钮旁边添加一个带图标的 toggle：
     - 开启（默认）: 「无头模式」灰色标签
     - 关闭: 「有头模式」高亮标签，表示浏览器可见

     样式与现有工具栏按钮一致，使用 border + rounded-lg + text-sm。

     验证

     1. 开启全局「无头模式」→ 运行含 GigaB2B/Amazon 节点的工作流 → 浏览器不弹出
     2. 切换为「有头模式」→ 运行 → 浏览器窗口弹出可见
     3. 节点配置面板的 headless 开关仍存在但被全局设置覆盖
